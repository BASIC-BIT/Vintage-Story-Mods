# Vintage Story Workspace

This repository expects local Vintage Story work to stay separated by source type: official game source, third-party mods, and our active working repositories.

## Recommended Layout

```text
<workspace-root>\
  external-mods\
    archives\        Downloaded third-party mod zip files.
    extracted\       Unpacked third-party mods.
    source-repos\    Cloned third-party mod repositories.
  source\
    vintagestory\
      <version>\
        downloads\   Official VS package downloads.
        bin\         Extracted official VS binaries/assets.
        decompiled\  Decompiled source projects generated by ilspycmd.
      legacy-unversioned\
        decompiled\  Previous unversioned source output, if retained locally.
  work\              Clean worktrees and additional working clones.
  Vintage-Story-Mods\ Active local repository/worktree.
```

The default maintainer workspace root is `C:\bench\vs`. Use `VS_WORKSPACE_ROOT` or pass `-WorkspaceRoot` to scripts when using another location.

## Refreshing Vintage Story Source

Use the repository script rather than manually downloading and decompiling game DLLs:

```powershell
.\scripts\Update-VintageStorySource.ps1 -WorkspaceRoot C:\bench\vs
```

For a specific version:

```powershell
.\scripts\Update-VintageStorySource.ps1 -Version 1.22.0 -WorkspaceRoot C:\bench\vs
```

To force regeneration:

```powershell
.\scripts\Update-VintageStorySource.ps1 -Version 1.22.0 -WorkspaceRoot C:\bench\vs -ForceExtract -ForceDecompile
```

The script downloads the official `windowsserver` package from Vintage Story's stable metadata, verifies the MD5 checksum, extracts it to `source\vintagestory\<version>\bin`, and decompiles these assemblies to `source\vintagestory\<version>\decompiled`:

- `VintagestoryAPI.dll`
- `VintagestoryLib.dll`
- `VintagestoryServer.dll`
- `Mods\VSSurvivalMod.dll`
- `Mods\VSEssentials.dll`
- `Mods\VSCreativeMod.dll`

## Agent Skill

The workspace procedure is also captured as an OpenCode skill at `.opencode\skills\vintage-story-workspace\SKILL.md`. Treat skills as first-class repository workflow assets: update the skill when the workflow changes, and include those changes in normal PR review.

## Guardrails

- Keep official downloads, extracted binaries, and decompiled source out of this repository.
- Prefer clean worktrees under `work\` when the active branch is dirty but current remote code is needed.
- Keep third-party mod packages and cloned external repos under `external-mods\`.
- Do not rewrite or move dirty repositories without explicit confirmation.
