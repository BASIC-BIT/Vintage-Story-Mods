# Pocket Waystone Prop Guide

This guide captures the durable modeling, design, and verification direction for the Pocket Waystone prop.

## Goal

Make `pocketdimensions:pocketwaystone` read as a useful in-game anchor without forcing one server lore. It should be distinctive enough that players recognize it as the pocket return/binding object, but neutral enough that servers can reframe it as a shrine, marker, terminal, altar, mirror anchor, rift device, elevator marker, or mundane admin utility.

DimensionLib remains unaware of this prop. The block, model, textures, name, recipe, interaction copy, and future chooser/portal behavior belong to Pocket Dimensions or a later utility layer.

## Recommended Tooling

Default model editor: Vintage Story Model Creator (VSMC).

Official tutorial reference: `https://wiki.vintagestory.at/Modding:Quick_Start_Tutorial`

Useful official reference pages:

- `https://wiki.vintagestory.at/Modding:Asset_System`
- `https://wiki.vintagestory.at/Modding:Block_Json_Properties`
- `https://apidocs.vintagestory.at/json-docs/jsondocs/Vintagestory.ServerMods.NoObf.BlockType.html`

VSMC is the best default because it edits native Vintage Story shape JSON and can preview textures with the same shape structure the game loads. The Quick Start Tutorial's modeling section documents setting a texture base path, opening shape JSON, editing cuboid elements, and checking UVs.

Blockbench can be useful as an alternative if VSMC is unavailable, using the community Vintage Story plugin, but treat VSMC/native JSON as the source of truth for this repo.

## Current First Pass

Current files:

- `assets/pocketdimensions/blocktypes/pocketwaystone.json`
- `src/PocketWaystoneBlock.cs`
- `src/PocketReturnPedestalBlock.cs`
- `src/PocketDimensionModSystem.cs`

The current blocktype is a low-risk first-pass JSON prop:

- `class: "PocketWaystoneBlock"`
- `drawtype: "json"`
- `shape: { base: "pocketdimensions:block/pocketwaystone" }`
- shape file: `assets/pocketdimensions/shapes/block/pocketwaystone.json`
- dedicated generated texture zones: `pocketwaystone-stone`, `pocketwaystone-trim`, and `pocketwaystone-accent`

The textures are generated by `scripts/generate-waystone-textures.ps1` so the first art pass is reproducible and easy to tune.

Externally placed Waystones persist their bound pocket id through `PocketWaystoneBlockEntity`. Generated in-pocket returns now use the separate `pocketdimensions:pocketreturnpedestal` block so the craftable ingress object and protected managed return object can evolve independently.

Keep the class registration in `PocketDimensionModSystem.Start(ICoreAPI api)`:

```csharp
api.RegisterBlockClass("PocketWaystoneBlock", typeof(PocketWaystoneBlock));
```

If this string does not match the JSON `class`, the game can load the block as a plain block and silently lose right-click behavior apart from log warnings.

- `assets/pocketdimensions/textures/block/pocketwaystone-accent.png`
- `assets/pocketdimensions/lang/en.json`

Reference paths:

- `shape: { base: "pocketdimensions:block/pocketwaystone" }` resolves to `assets/pocketdimensions/shapes/block/pocketwaystone.json`.
- `pocketdimensions:block/pocketwaystone-stone` resolves to `assets/pocketdimensions/textures/block/pocketwaystone-stone.png`.

Avoid old `dimensionpockets` asset paths. The released mod id/domain is `pocketdimensions`.

## Visual Direction

First real model: compact neutral anchor plinth.

Recommended silhouette:

- Low plinth base.
- Narrower upright core.
- Recessed front plate for label/destination/configuration affordance.
- Small top socket or inset plate as the activation focus.
- No large portal ring yet.

Recommended scale:

- Height: `0.65` to `0.85` blocks.
- Footprint: `0.55` to `0.75` blocks wide/deep.
- Collision: simple cuboid slightly smaller than visual footprint.
- Selection: large enough to target comfortably, but not a full-block lie if the model is visibly small.

Recommended complexity:

- 6 to 10 cuboids.
- Mostly rectangular cuboids.
- 1 or 2 bevel-like steps.
- No animation in the first real pass.
- No dependence on particles or glow for identity.

Recommended material zones:

- Base material: neutral stone/slate/clay/metal/wood composite.
- Trim material: muted brass/bronze/darker stone.
- Accent material: small blue, green, amber, or violet inlay/socket.

Avoid default runes, holy symbols, occult symbols, faction marks, demonic marks, sci-fi holograms, and huge obelisks. Those lock the prop to a server fantasy too early.

## Blocktype Pattern

This is the recommended blocktype shape pattern for the prop:

```json
{
  "code": "pocketwaystone",
  "class": "PocketWaystoneBlock",
  "blockmaterial": "Stone",
  "drawtype": "json",
  "shape": { "base": "pocketdimensions:block/pocketwaystone" },
  "creativeinventory": { "general": ["*"], "decorative": ["*"] },
  "textures": {
    "stone": { "base": "pocketdimensions:block/pocketwaystone-stone" },
    "accent": { "base": "pocketdimensions:block/pocketwaystone-accent" }
  },
  "sideopaque": { "all": false },
  "sideao": { "all": false },
  "sidesolid": { "all": false },
  "collisionbox": { "x1": 0.2, "y1": 0, "z1": 0.2, "x2": 0.8, "y2": 0.8, "z2": 0.8 },
  "selectionbox": { "x1": 0.15, "y1": 0, "z1": 0.15, "x2": 0.85, "y2": 0.95, "z2": 0.85 },
  "replaceable": 0,
  "resistance": 31337,
  "requiredMiningTier": 99,
  "lightAbsorption": 0,
  "drops": []
}
```

Set `sideopaque`, `sideao`, and `sidesolid` false for non-cube props. This prevents full-cube face culling, ambient occlusion, and support-face assumptions.

## Shape Pattern

Minimal shape structure:

```json
{
  "textureWidth": 32,
  "textureHeight": 32,
  "textures": {
    "stone": "pocketdimensions:block/pocketwaystone-stone",
    "accent": "pocketdimensions:block/pocketwaystone-accent"
  },
  "elements": [
    {
      "name": "Base",
      "from": [3, 0, 3],
      "to": [13, 3, 13],
      "faces": {
        "north": { "texture": "#stone", "uv": [0, 0, 10, 3] },
        "east": { "texture": "#stone", "uv": [0, 0, 10, 3] },
        "south": { "texture": "#stone", "uv": [0, 0, 10, 3] },
        "west": { "texture": "#stone", "uv": [0, 0, 10, 3] },
        "up": { "texture": "#stone", "uv": [0, 0, 10, 10] },
        "down": { "texture": "#stone", "uv": [0, 0, 10, 10] }
      }
    }
  ]
}
```

Prefer creating the final shape in VSMC and committing the generated JSON. Keep names meaningful: `Base`, `Core`, `FrontPlate`, `TopSocket`, `AccentInlay`.

## Verification Checklist

Package checks:

- Confirm the zip is `pocketdimensions_0_1_0.zip`, not an old `dimensionpockets_0_1_0.zip`.
- Confirm the zip contains `assets/pocketdimensions/blocktypes/pocketwaystone.json`.
- Confirm the zip contains `assets/pocketdimensions/blocktypes/pocketreturnpedestal.json`.
- Confirm the zip contains the shape JSON referenced by `shape.base`.
- Confirm the zip contains every referenced texture PNG.
- Confirm `modinfo.json` still uses `modid: pocketdimensions`.

Server log checks:

- Confirm `Mod 'pocketdimensions_0_1_0.zip' (pocketdimensions)` loads.
- Search for `pocketwaystone`, `pocketreturnpedestal`, `shape`, `texture`, `not found`, `errored`, `Exception`, `Error`, and `WARNING`.
- Treat missing `PocketWaystoneBlock` class registration as a blocker.
- Treat missing `PocketReturnPedestalBlock` class registration as a blocker.

Client log checks:

- Confirm server assets load.
- Search for missing shape/texture lines.
- Confirm no fallback white/red texture behavior.
- Confirm no asset domain mismatch such as `dimensionpockets:`.

In-game verification:

- Use `/pocket create <name>` to refresh/materialize the pocket.
- Use `/pocket inspect <name>` to confirm the expected return pedestal position and actual block code.
- Craft or place an external `pocketdimensions:pocketwaystone`.
- Look at the external Waystone and run `/pocket bind <name>`.
- Right-click the bound external Waystone and confirm it enters the bound pocket.
- Break the external Waystone and confirm it drops as an item.
- Place or inspect the block in creative mode when manual QA is approved.
- Confirm selection outline matches `selectionbox`.
- Confirm collision matches `collisionbox`.
- Confirm adjacent faces are not culled incorrectly.
- Confirm right-clicking the return pedestal returns to the linked external Waystone endpoint after Waystone entry and to the captured command-entry point after `/pocket enter`.
- Confirm player breaking is denied for `pocketdimensions:pocketreturnpedestal`.

Do not mark the prop done based only on "looks okay." The prop needs clean logs, correct package contents, correct block code at the managed return location, correct selection/collision behavior, and retained right-click behavior.

## Staged Roadmap

Stage 1: placeholder cube.

Stage 2: neutral anchor plinth with real JSON shape using the existing floor texture.

Stage 3: dedicated generated texture zones for stone, trim, and accent materials.

Stage 4: craftable, breakable, bindable external Waystones.

Stage 5: optional visual variants such as unbound, bound, locked, and unavailable.

Stage 6: server customization for display name, texture pack, accent color, and recipe.

Stage 7: portal feedback and chooser UI after the Waystone interaction flow is stable.
